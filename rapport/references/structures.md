# Structures détaillées par type de rapport

Ce fichier contient les plans de référence pour chaque type de rapport. Le skill doit consulter ce fichier pour adapter le plan au type détecté.

---

## 1. Compte-rendu de TP

### Plan type

```
Page de garde
Sommaire
1. Introduction
   1.1 Contexte et objectifs du TP
   1.2 Rappels théoriques (résumé ciblé, pas de copie de cours)
2. Matériel et méthodes
   2.1 Matériel utilisé (machines, instruments, logiciels)
   2.2 Protocole expérimental (étapes numérotées)
   2.3 Conditions d'essai (paramètres, réglages)
3. Résultats
   3.1 Résultats bruts (tableaux, mesures)
   3.2 Résultats traités (graphiques, calculs)
4. Analyse et interprétation
   4.1 Comparaison résultats / théorie
   4.2 Sources d'erreur et incertitudes
   4.3 Discussion
5. Conclusion
Annexes (données brutes, code, photos)
```

### Points d'attention
- Les rappels théoriques doivent être CIBLÉS : uniquement la théorie mobilisée dans le TP, pas un cours entier
- Protocole : assez détaillé pour qu'un autre étudiant puisse reproduire le TP
- Résultats : séparer données brutes (tableaux) et données traitées (graphiques)
- Analyse : TOUJOURS comparer aux valeurs théoriques si elles existent
- Calculer les écarts relatifs : |Vexpérimental - Vthéorique| / Vthéorique × 100%
- Identifier et discuter les sources d'erreur (systématiques vs aléatoires)

## 2. Rapport de SAÉ (Situation d'Apprentissage et d'Évaluation)

### Plan type

```
Page de garde (titre SAÉ, membres du groupe, tuteur, dates)
Sommaire
1. Introduction
   1.1 Contexte et commanditaire
   1.2 Cahier des charges / Objectifs
   1.3 Organisation du groupe (rôles, planning)
2. Analyse du besoin
   2.1 Étude de l'existant
   2.2 Contraintes (techniques, économiques, temporelles)
   2.3 Spécifications fonctionnelles
3. Conception / Démarche
   3.1 Recherche de solutions
   3.2 Critères de choix
   3.3 Solution retenue et justification
4. Réalisation / Mise en œuvre
   4.1 Étapes de réalisation
   4.2 Difficultés rencontrées et solutions
   4.3 Résultats obtenus
5. Bilan
   5.1 Confrontation résultats / cahier des charges
   5.2 Compétences mobilisées (référentiel BUT)
   5.3 Apport personnel et retour d'expérience
6. Conclusion et perspectives
Annexes (plans, code, devis, photos, planning Gantt)
Bibliographie / Webographie
```

### Points d'attention
- **Compétences BUT** : mapper explicitement les compétences du référentiel BUT GMP aux actions réalisées
- **Travail de groupe** : mentionner la répartition des tâches, la gestion de projet
- **Démarche CPD** (Conception de Produits Durables) si parcours CPD : intégrer les aspects éco-conception, cycle de vie, matériaux durables
- **Budget** : si applicable, présenter un tableau des coûts
- **Planning** : inclure un Gantt ou un chronogramme en annexe
- **Photos** : documenter les étapes de réalisation avec des photos légendées

## 3. Rapport de stage / alternance

### Plan type

```
Page de garde
Remerciements
Sommaire
Liste des figures et tableaux
Glossaire / Liste des abréviations
1. Introduction
2. Présentation de l'entreprise
   2.1 Historique et secteur d'activité
   2.2 Organisation et organigramme
   2.3 Produits / services
   2.4 Position dans le service d'accueil
3. Missions réalisées
   3.1 Mission principale
      3.1.1 Contexte et problématique
      3.1.2 Méthodologie
      3.1.3 Résultats
   3.2 Missions secondaires (même structure)
4. Bilan technique et personnel
   4.1 Compétences techniques acquises
   4.2 Compétences transversales développées
   4.3 Lien avec la formation (compétences BUT)
   4.4 Difficultés et apprentissages
5. Conclusion
   5.1 Synthèse
   5.2 Perspectives professionnelles
Bibliographie
Annexes
Résumé / Abstract (4ème de couverture)
```

### Points d'attention
- **Confidentialité** : vérifier si certaines données sont confidentielles
- **Remerciements** : tuteur entreprise, tuteur académique, collègues
- **Organigramme** : situer sa position dans l'entreprise
- **Missions** : décrire le CONTEXTE avant la méthode, la méthode avant les résultats
- **Bilan** : être honnête sur les difficultés, c'est valorisé
- **Compétences BUT** : obligatoire en BUT, mapper avec le référentiel
- **Résumé** : français + anglais, 150-250 mots, avec mots-clés
- **Longueur typique** : 30-50 pages hors annexes (BUT3), 20-30 pages (BUT2)

## 4. Dossier technique / Étude technique

### Plan type

```
Page de garde
Sommaire
1. Introduction
   1.1 Contexte du projet
   1.2 Cahier des charges fonctionnel
2. Analyse fonctionnelle
   2.1 Bête à cornes, diagramme pieuvre
   2.2 Fonctions de service et contraintes
   2.3 Hiérarchisation des fonctions
3. Étude de conception
   3.1 Recherche de solutions (TRIZ, brainstorming, benchmark)
   3.2 Analyse multicritère
   3.3 Solution retenue
4. Dimensionnement
   4.1 Calculs (RDM, thermique, cinématique...)
   4.2 Choix des matériaux
   4.3 Choix des composants standards
5. Modélisation / Plans
   5.1 Modèle CAO (captures SolidWorks/CATIA)
   5.2 Mise en plan (cotation fonctionnelle)
   5.3 Nomenclature
6. Industrialisation (si applicable)
   6.1 Gamme de fabrication
   6.2 Analyse des coûts
   6.3 Contrôle qualité
7. Conclusion
Annexes (plans détaillés, notes de calcul, fiches matériaux)
Bibliographie
```

### Points d'attention
- **Cahier des charges** : fonctionnel, pas seulement technique
- **Analyse fonctionnelle** : utiliser les outils normalisés (NF X50-151)
- **Choix** : TOUJOURS justifier avec une matrice de décision pondérée
- **Calculs** : présenter la démarche, pas juste le résultat
- **CAO** : captures d'écran annotées, pas juste des vues ISO
- **Cotation** : fonctionnelle, pas de surcotation
- **Éco-conception** (CPD) : analyse du cycle de vie, choix matériaux durables

## 5. Note de synthèse

### Plan type

```
Titre et références
1. Objet de la synthèse
2. Points essentiels
   2.1 Fait / constat principal
   2.2 Analyse
   2.3 Points d'attention
3. Recommandations / Décisions
Annexes (si nécessaire)
```

### Points d'attention
- **Concision** : 2-4 pages maximum
- **Structure** : chaque paragraphe = une idée = une information
- **Pas d'opinion personnelle** sauf si explicitement demandé
- **Données chiffrées** : privilégier les faits aux impressions
- **Destinataire** : adapter le niveau technique au lecteur

## Règles de mise en forme transversales

### Pagination
- Page de garde : pas de numéro
- Sommaire : chiffres romains (i, ii, iii)
- Corps : chiffres arabes (1, 2, 3...)

### Police et tailles
- Corps : 11-12pt, police lisible (Calibri, Arial, Times New Roman)
- Titres : hiérarchie claire (16pt > 14pt > 12pt gras)
- Interligne : 1.15 à 1.5
- Marges : 2.5cm minimum

### Figures et tableaux
- Numérotation séparée (Figure 1, Figure 2... / Tableau 1, Tableau 2...)
- Légende sous les figures, au-dessus des tableaux
- Référencées dans le texte : "Comme le montre la Figure 3..."
- Lisibles en noir et blanc (pas que de la couleur pour différencier)

### Citations et bibliographie
- Format APA ou Vancouver (cohérent dans tout le document)
- Toute affirmation non triviale doit être sourcée
- Webographie : URL + date de consultation
